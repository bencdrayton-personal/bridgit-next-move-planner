# Roadmap — Next Move Planner

Outcome-based roadmap: each phase commits to a question answered, not a feature list shipped. Features are the current best bet for answering it.

## Now (0–3 months) — "Do homeowners want this, and do they trust it?"

| Bet | Measure |
|---|---|
| Production-harden the prototype (analytics, accessibility, legal copy) | Compliance sign-off; WCAG 2.1 AA |
| Soft launch behind 2–3 SEO content pages ("can I buy before I sell?") | ≥55% assessment completion; organic CTR |
| Run discovery experiments E1–E5 (fake door, usability × 5 personas, broker panel) | Assumptions A1–A5 validated/killed |
| Instrument the full funnel | Event spec live; weekly North Star reporting |

**Kill criteria:** completion < 30% after copy iteration, or broker panel rejects scoring credibility → revisit core framing before scaling.

## Next (3–9 months) — "Does readiness convert to pipeline?"

| Bet | Measure |
|---|---|
| Broker-conversation CTA → booking flow integration | ≥8% CTA conversion; show-rate of booked conversations |
| Suburb-level days-on-market data grounding scenarios | Scenario engagement ↑; qualitative trust signals |
| Saved plans + re-check nudges ("your 6-month review") | Return rate ≥15%; "not-yet" cohort reactivation |
| Valuation API integration (estimate ranges replace single guesses) | Confidence-score realism; complaint rate stays ~0 |
| AI narrative v2: personalised guidance within guardrails | Eval-set pass rate; zero compliance escalations |

## Later (9–18 months) — "Can the planner become distribution?"

| Bet | Measure |
|---|---|
| Consent-based plan handoff to brokers (summary arrives before the fact-find) | Broker NPS on handoffs; fact-find time saved |
| White-label planner pilot with one aggregator partner | Pilot signed; assessments via partner channel |
| Readiness signals pre-filling Bridgit applications (with consent) | Application completion time ↓; approval rate ↑ |
| Post-settlement loop: "your plan vs what happened" | Data asset for scoring calibration; advocacy content |

## Sequencing logic

1. **Trust before conversion.** Now-phase optimises honesty and completion, deliberately not referral volume — converting distrust early poisons the well.
2. **Conversion before distribution.** White-labelling an unproven funnel exports a question mark. Next-phase proves the unit economics on Bridgit's own channel first.
3. **Data compounds.** Every phase adds a data layer (behavioural → market → valuation → outcome) that makes the readiness verdict more accurate and harder to copy.

## Explicitly deferred

- Conversational AI mode — deferred until guardrail design + eval harness exist (see ai-opportunities.md)
- Property search / listings integration — partner surface, not core; revisit at H3
- Non-bridging product recommendations — strategic distraction; the planner wins by owning one question
